Detailed Description

EFR32xG1x Specific Interrupt sources.

Below are the interrupt handlers implemented within the RAIL library for EFR32. Please see EFR32 for more details about interrupt initialization and priorities.

Function Documentation

void AGC_IRQHandler ( void  )

RAIL interrupt handler for the AGC interrupt source.

This source handles interrupts generated from the automatic gain control block. The following events are currently in use:

  • RSSI Valid
    • This event is enabled by RAIL_StartAverageRSSI() and occurs when the first RSSI sample is ready after going into RX state.
void BUFC_IRQHandler ( void  )

RAIL interrupt handler for the BUFC interrupt source.

This source handles interrupts generated from the radio buffer controller. The following events are currently in use:

  • RX Buffer Threshold
    • This event is enabled while using fifo mode. In fifo mode, this event will generate RAILCb_RxFifoAlmostFull() when the contents of the receive buffer exceed the receive threshold set by RAIL_SetRxFifoThreshold(). This event is also enabled and consumed within RAIL when configured for BER mode via RAIL_BerConfigSet().
  • RX Buffer Underflow
    • This event cannot be disabled and indicates too much data was read from the RX buffer. This event will generate RAILCb_RxRadioStatusExt() - RAIL_RX_CONFIG_BUFFER_UNDERFLOW.
  • RX Buffer Corruption
    • This event cannot be disabled and indicates the pointers in the RX buffer have become corrupt. This event should not occur in normal operation.
  • TX Buffer Threshold
    • This event is enabled while using fifo mode. In fifo mode, this event will generate RAILCb_TxFifoAlmostEmpty() when the contents of the transmit buffer falls beneath the transmit threshold set by RAIL_SetTxFifoThreshold(). When the transmit buffer contents exceeds the transmit threshold, this interrupt will fire due to its edge based implmentation but will not cause a callback.
  • TX Buffer Overflow
    • This event cannot be disabled and indicates too much data was placed into the TX buffer. This event will generate RAILCb_TxRadioStatus() - RAIL_TX_CONFIG_BUFFER_OVERFLOW.
  • TX Buffer Corruption
    • This event cannot be disabled and indicates the pointers in the TX buffer have become corrupt. This event should not occur in normal operation.

This interrupt vector will also fire on every valid packet received event.

void FRC_IRQHandler ( void  )

RAIL interrupt handler for the FRC interrupt source.

This source handles most packet completion and error events. The following events are currently in use:

  • Rx Aborted
    • This event cannot be disabled and will generate RAILCb_RxRadioStatusExt() - RAIL_RX_CONFIG_PACKET_ABORTED when RX is exited while a packet is being received. This can occur due to RAIL_RfIdle() or a filtered packet.
  • RX Frame Error
    • This event cannot be disabled and will generate RAILCb_RxRadioStatusExt() - RAIL_RX_CONFIG_FRAME_ERROR when a packet is received with a bad CRC or invalid length field.
  • RX Buffer Overflow
    • This event cannot be disabled and will generate RAILCb_RxRadioStatusExt() - RAIL_RX_CONFIG_BUFFER_OVERFLOW when a receive buffer overflow occurs.
  • RX Complete
    • This event cannot be disabled and will generate RAILCb_AllocateMemory(), RAILCb_BeginWriteMemory(), RAILCb_EndWriteMemory(), RAILCb_RxPacketReceived(), and RAILCb_FreeMemory() when a packet is received successfully. This interrupt will remain pending until all received packets are processed.
  • TX Aborted
    • This event cannot be disabled and will generate RAILCb_TxRadioStatus() - RAIL_TX_CONFIG_TX_ABORTED when a transmit is aborted.
  • TX Buffer Underflow
    • This event cannot be disabled and will generate RAILCb_TxRadioStatus() - RAIL_TX_CONFIG_BUFFER_UNDERFLOW if the transmit buffer underflows.
  • TX Complete
    • This event cannot be disabled and will generate RAILCb_TxPacketSent() when a transmit is completed successfully.
void FRC_PRI_IRQHandler ( void  )

RAIL interrupt handler for the FRC_PRI interrupt source.

This source handles high priority interrupt events within the FRC. Currently there are no interrupt sources enabled within this group.

Note
The interrupt vector should point to the RAIL interrupt handler for compatibility with future RAIL releases.
void MODEM_IRQHandler ( void  )

RAIL interrupt handler for the MODEM interrupt source.

This source handles demodulator and modulator status and error events. The following events are currently in use:

  • Preamble Detection
    • This event is enabled by RAIL_RxConfig() when provided RAIL_RX_CONFIG_PREAMBLE_DETECT. This event will generate RAILCb_RxRadioStatus() - RAIL_RX_CONFIG_PREAMBLE_DETECT when a preamble is detected by the demodulator and the callback is enabled.
  • Sync 1 Detection
    • This event is enabled by RAIL_RxConfig() when provided RAIL_RX_CONFIG_SYNC1_DETECT. This event will generate RAILCb_RxRadioStatus() - RAIL_RX_CONFIG_SYNC1_DETECT when Sync 1 is detected by the demodulator and the callback is enabled.
  • Sync 2 Detection
    • This event is enabled by RAIL_RxConfig() when provided RAIL_RX_CONFIG_SYNC2_DETECT. This event will generate RAILCb_RxRadioStatus() - RAIL_RX_CONFIG_SYNC2_DETECT when Sync 2 is detected by the demodulator and the callback is enabled.
void PROTIMER_IRQHandler ( void  )

RAIL interrupt handler for the PROTIMER interrupt source.

This source handles interrupts generated from the RAIL timebase (PROTIMER). The following events are currently in use:

  • Scheduled TX/RX
    • This event cannot be disabled and occurs when a scheduled transmit or receive begins.
  • RX timeout scheduled window end
    • This event is enabled by RAIL_ScheduleRx() and will generate RAILCb_RxRadioStatusExt() - RAIL_RX_CONFIG_SCHEDULED_RX_END when a frame is not received within the scheduled window.
  • Transmit with CCA Failed
    • This event cannot be disabled and will generate RAILCb_TxRadioStatus() - RAIL_TX_CONFIG_CHANNEL_BUSY when a transmit using CCA has failed.
  • Transmit with CCA Success
    • This event cannot be disabled and occurs when CCA indicates the channel is clear and the packet will be transmitted. This event will generate RAILCb_TxRadioStatus() - RAIL_TX_CONFIG_CHANNEL_CLEAR.
  • Starting a CCA check
    • This event is enabled by RAIL_TxConfig() and occurs when the radio begins warming up for a CCA check. This event will generate RAILCb_TxRadioStatus() - RAIL_TX_CONFIG_START_CCA.
  • CCA Retry
    • This event is enabled by RAIL_TxConfig() and occurs when CSMA or LBT get a failed CCA and begin retrying. This event will generate RAILCb_TxRadioStatus() - RAIL_TX_CONFIG_CCA_RETRY.
  • RAIL Timer
    • This event is enabled by RAIL_TimerSet() and will generate RAILCb_TimerExpired() when the timer expires.
void RAC_RSM_IRQHandler ( void  )

RAIL interrupt handler for the RAC_RSM interrupt source.

This source handles interrupts generated from the radio controller state. The following events are currently in use:

  • PA Voltage High
    • This event cannot be disabled and occurs once per transmit while the PA voltage is over the max rating.
  • PA Output High
    • This event cannot be disabled and occurs while the PA output power is over the max rating.
  • PA Output Low
    • This event cannot be disabled and occurs once per transmit while the PA output power is below the configured level.
  • Radio State Change
    • This event is enabled by RAIL_DebugCbConfig() when provided RAIL_DEBUG_CONFIG_STATE_CHANGE. This event will generate RAILCb_RadioStateChanged() whenever the state of the radio hardware changes.
void RAC_SEQ_IRQHandler ( void  )

RAIL interrupt handler for the SEQ interrupt source.

This source handles interrupts generated from the radio sequencer. The following events are currently in use:

  • Address Filtered
    • This event is enabled when using Address_Filtering and will generate RAILCb_RxRadioStatusExt() - RAIL_RX_CONFIG_ADDRESS_FILTERED when a packet is address filtered.
  • Ack Timeout
    • This event is enabled when using Auto_Ack and will generate RAILCb_RxAckTimeout() when a ack packet is not received within the ACK window.
  • IEEE802154 Frame Pending
    • This event is enabled by RAIL_IEEE802154_Init() and will generate RAILCb_IEEE802154_DataRequestCommand() when a valid IEEE802154 data request packet is received.
  • Temperature Calibration
    • This event cannot be disabled and will generate RAILCb_CalNeeded() when the temperature changes enough to require recalibration during RX state.
void RFSENSE_IRQHandler ( void  )

RAIL interrupt handler for the RFSENSE interrupt source.

This source handles interrupts generated from the RF Sense module. The following events are currently in use:

  • RF Sensed
    • This event is enabled by when the RFSENSE block is initialized using RAIL_RfSense() and can be disabled by turning off the RFSENSE block using RAIL_RfSense() with parameter RAIL_RFSENSE_OFF. This event will possibly generate RAILCb_RxRadioStatusExt() - RAIL_RX_CONFIG_RF_SENSED whenever the RFSENSE block detects RF energy.
