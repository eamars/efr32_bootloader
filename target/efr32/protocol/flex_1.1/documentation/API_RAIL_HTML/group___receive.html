bool RAIL_AverageRSSIReady ( void  )

Queries whether the RSSI averaging is done.

Returns
Returns true if done and false otherwise.

This function can be used to poll for completion of the RSSI averaging so that you do not have to rely on an interrupt based callback.

int16_t RAIL_GetAverageRSSI ( void  )

Get the RSSI averaged over specified time in us.

Returns
Return RAIL_RSSI_INVALID if the receiver is disabled and we are unable to get an RSSI value, otherwise, return the RSSI in quarter dBm, dbm*4.

Get the hardware RSSI average after issuing RAIL_StartAverageRSSI. Should be used after RAIL_StartAverageRSSI.

int16_t RAIL_PollAverageRSSI ( uint32_t  averageTimeUs)

Compute the average RSSI over a specified time in us.

Parameters
[in]averageTimeUsAveraging time in microseconds.
Returns
Return RAIL_RSSI_INVALID if the receiver is disabled and we are unable to get an RSSI value, otherwise, return the RSSI in quarter dBm, dbm*4.

This blocking function will poll the hardware for RSSI values and compute the average RSSI over the requested time period. If no valid readings have been made function will return RAIL_RSSI_INVALID reading. Receiving a packet during the averaging will cause invalid reading(s). However, invalid readings during the averaging will not be included in the average. Number of RSSI readings per baud depends on the phy.

int16_t RAIL_RxGetRSSI ( void  )

Return the current raw RSSI.

Returns
Return RAIL_RSSI_INVALID if the receiver is disabled and we are unable to get an RSSI value, otherwise, return the RSSI in quarter dBm, dbm*4.

Get the current RSSI value. This value represents the current energy of the channel, so it can change rapidly, and will be low if there is no RF energy in your current channel. The function from the value reported to dBm is an offset dependent on the PHY and the PCB layout. Users should characterize the RSSI received on their hardware and apply an offset in the application to account for board and PHY parameters.

uint8_t RAIL_ScheduleRx ( uint8_t  channel,
RAIL_ScheduleRxConfig_t *  cfg 
)

Schedule a receive window for some time in the future.

Parameters
[in]channelChannel to listen on
[in]cfgThe configuation struct to define the receive window.
Returns
Return 0 on success or an error code

This API will immediately change your channel and schedule receive to start at the specified time and end at the given end time. If you do not specify an end time then you may call this API later with an end time as long as you set the start time to disabled. You can also terminate the whole receive operation immediately using the RAIL_RfIdle() function. Note that relative end times are always relative to the start unless there is no start time specified.

Callback for when AGC averaged RSSI is done.

Parameters

void RAILCb_RxPacketReceived ( void *  rxPacketHandle)

Receive packet callback.

Parameters
[in]rxPacketHandleContains a handle that points to the memory that the packet was stored in. This handle will be the same as something returned by the RAILCb_AllocateMemory() API. This handle will hold a RAIL_RxPacketInfo_t structure starting at offset 0 in the buffer.

This function is called whenever a packet is received and returns to you the memory handle for where this received packet and its appended information was stored. After this callback is done we will release the memory handle so you must somehow increment a reference count or copy the data out within this function.

If RAIL_IGNORE_CRC_ERRORS is set, this callback will fire for packets with crc errors as well.

void RAILCb_RxRadioStatus ( uint8_t  status)

Called whenever an enabled radio status event occurs.

Parameters
[in]statusThe event that triggered this callback

The triggers that cause this function to be called can be enabled using the RAIL_RxConfig() function.

Note
This function will return only the first 8 of all possible triggers. For accessing all triggers see the new RAILCb_RxRadioStatusExt() API. If you implement RAILCb_RxRadioStatusExt() this callback will no longer be used by the RAIL library. In RAIL 2.0 this API will be merged with the RAILCb_RxRadioStatusExt() for one clean interface.

Triggers:

  • RAIL_RX_CONFIG_PREAMBLE_DETECT
  • RAIL_RX_CONFIG_SYNC1_DETECT
  • RAIL_RX_CONFIG_SYNC2_DETECT
  • RAIL_RX_CONFIG_FRAME_ERROR
  • RAIL_RX_CONFIG_BUFFER_OVERFLOW
  • RAIL_RX_CONFIG_ADDRESS_FILTERED
  • RAIL_RX_CONFIG_RF_SENSED
void RAILCb_RxRadioStatusExt ( uint32_t  status)

Called whenever an enabled radio status event occurs.

Parameters
[in]statusThe event or events that triggered this callback

The triggers that cause this function to be called can be enabled using the RAIL_RxConfig() function. This function is the same as RAILCb_RxRadioStatus() with an extended set of triggers. For backwards compatibility this function is weakly defined in the RAIL library to call RAILCb_RxRadioStatus() with the subset of valid events. If you need more events you must implement this version which will stop the old one from being called.

Note
In RAIL 2.0 this API will be merged with the RAILCb_RxRadioStatus() for one clean interface.

Triggers:

  • RAIL_RX_CONFIG_PREAMBLE_DETECT
