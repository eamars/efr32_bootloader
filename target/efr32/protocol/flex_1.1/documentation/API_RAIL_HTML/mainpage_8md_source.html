1 RAIL Library {#mainpage}
2 ============
3 The Silicon Labs Radio Abstraction Interface Layer (RAIL) is a library that can
4 be used as a generic interface for all Silicon Labs radio parts. By programming
5 against this API you are able to write code that easily ports across different
6 radio parts while having access to hardware acceleration wherever possible.
7 
8 # Introduction
9 
10 The RAIL library is as standalone as it can be with few external dependencies.
11 To build applications using it we recommend using Simplicity Studio's
12 application builder since it will simplify the process but that is not required.
13 Depending on the hardware platform that you're using you may be required to
14 provide certain HAL functionality for the library to work properly. These
15 functions should be covered in the section of the guide for your hardware
16 platform and valid implementations of them can be found as a part of the
17 emlib/emdrv HAL layers that we provide. It is recommended that you use these
18 versions for maximum support, but you're free to re-implement them if necessary.
19 
20 # Using RAIL
21 
22 ## Features
23 
24 At a high level the functionality supported by RAIL is shown below. For more
25 specifics on the APIs and how to use them see the module documentation.
26 
27 * \ref General
28  * Initialize the RAIL API layer and perform basic radio state transitions.
29  * Configure automatic radio state transitions and transition timings
30  * Enable RF energy sensing of specified duration across the 2.4 GHz and/or
31  Sub-GHz bands (EFR32 only).
32 * \ref Radio_Configuration
33  * Configure the radio frequency, packet format, channel configuration and
34  other PHY parameters.
35  * Query current PHY data rates and parameters like current channel.
36 * \ref Memory_Management
37  * Integrate RAILs memory management callbacks with your upper layer to give
38  us some place to put a received packet.
39 * \ref Data_Management
40  * Allows the application to choose the type of data and the method of
41  data interaction through RAIL.
42 * \ref Receive
43  * Start receive.
44  * Configure receive status callbacks.
45  * Enable and configure \ref Address_Filtering for each packet.
46 * \ref Transmit
47  * Load and send packet data.
48  * Configure transmit status callbacks.
49  * Configure advanced \ref Pre-Transmit hooks for scheduled transmit, CSMA, and
50  LBT.
51 * \ref System_Timing
52  * Get the current time in the RAIL timebase.
53  * Configure a timer to call you back after an absolute or relative delay.
54 * \ref Auto_Ack
55  * Configure the radio for automatic acknowledgements
56  * Load the auto ack payload
57 * \ref Diagnostic
58  * Output debug signals like an unmodulated tone and a continuously modulated
59  stream of data.
60  * Configure crystal tuning for your radio.
61  * Bit error rate testing functionality
62 * \ref Calibration
63  * APIs for handling various radio calibrations for optimal performance.
64 
65 Protocol specific hardware acceleration:
66 
67 * \ref IEEE802_15_4
68  * Configure the IEEE802.15.4 2.4GHz PHY
69  * Configure node address and address filtering for IEEE 802.15.4
70  * Configure auto ack for IEEE 802.15.4
71 * \ref BLE
72  * Configure the Bluetooth Low Energy 1Mbit PHY
73  * Preamble, sync word and whitening adjustment function for connections

75 ## Getting Started Example
76 
77 Below is a simple example to show you how to initialize the RAIL library in
78 your application. Notice that this requires radio, frame, and channel
79 configurations which can be generated from the radio calculator in Simplicity
80 Studio. By default RAIL interacts with data on a per-packet basis; for more
81 information refer to \ref Data_Management.
82 
83 _Note: Before initializing the radio, make sure the PA is initalized for your
84 platform. This is covered in the Hardware Specific Configuration section below_
85 
86 ~~~~~~~~~~~~~~~{.c}
87 static const RAIL_Init_t railInitParams = {
88  APP_MAX_PACKET_LENGTH, // UNUSED - Will be removed in a future RAIL version
89  RAIL_RF_XTAL_FREQ, // in Hz, so this is 38400000 on the EFR32
90 };
91 
92 static const RAIL_CalInit_t railCalInitParams = {
93  RAIL_CAL_ALL, // Run all possible calibrations for this chip
94  irCalConfig, // Settings specific to IR calibration
95 };
96 
97 // Initialize the radio out of startup so that it's ready to receive
98 int radioInitialize(void)
99 {
100  // Initialize the RAIL library and any internal state it requires
101  RAIL_RfInit(&railInitParams);
102 
103  // Initialize calibration settings
104  RAIL_CalInit(&railCalInitParams);
105 
106  // Apply the selected RADIO configuration
107  if (RAIL_RadioConfig((void*)configList[0])) {
108  // Error: Could not apply the radio configuration
109  while(1);
110  }
111 
112  // Set us to a valid channel for this config
113  RAIL_ChannelConfig(channelConfigs[0]);
114 
115  // Configure a few error callbacks and enable appended info in the returned
116  // output structure
117  RAIL_RxConfig(( RAIL_RX_CONFIG_INVALID_CRC
118  | RAIL_RX_CONFIG_ADDRESS_FILTERED),
119  true);
120 
121  // Set automatic transitions to always receive once started
122  RAIL_SetRxTransitions(RAIL_RF_STATE_RX, RAIL_RF_STATE_RX,
123  RAIL_IGNORE_NO_ERRORS);
124  RAIL_SetTxTransitions(RAIL_RF_STATE_RX, RAIL_RF_STATE_RX);
125 }
126 ~~~~~~~~~~~~~~~
