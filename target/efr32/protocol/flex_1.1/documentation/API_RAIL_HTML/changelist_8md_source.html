4 ### 1.5.2
5  * Fix an issue in 1.5.1 that would cause scheduled receive, scheduled
6  transmit, and RAIL timer operations to fail on EFR32xG12 with a relative
7  delay of 0.
8  * Fix an issue in 1.5.1 that would cause scheduled receive and scheduled
9  transmit to fail if called with an absolute time from [0, rxWarmTime) or
10  [0, txWarmTime) respectively.
11  * Fix an issue where a failed scheduled transmit operation would prevent any
12  transmit from succeeding until \ref RAIL_RfIdle() or \ref RAIL_RfIdleExt()
13  were called.
14  * Fix an issue in 1.5.1 where calling idle while looking for an ACK could
15  cause the radio to get stuck in the receive state.
16 
17 ### 1.5.1
18  * Add official support for the EFR32xG12 parts through a new
19  librail_efr32xg12.a version of the library.
20  * Added RAIL_SetRxOptions API for configuration of receive options, such as
21  RAIL_RX_OPTION_STORE_CRC.
22  * Increase transmit and receive fifo sizes to 512 bytes for all EFR32xG1
23  parts. This impacts both packet and fifo modes.
24  * Added \ref RAIL_RxDataSource_t::RX_DEMOD_DATA in order to receive
25  demodulated data via FIFO method.
26  * Shortened the duration that the transmit buffer lock in RAIL_TxDataLoad()
27  is held, as it was overprotective. RAIL will still report an error if the
28  application attempts to manipulate the transmit buffer while actively
29  transmitting data.
30  * Improved state transition timings for enhanced precision and accuracy. To
31  improve precision we have switched to a more stable receive complete event
32  since our previous trigger could jitter for certain radio configurations. To
33  improve accuracy we measured and removed PHY specific overhead for BLE
34  applications. Note that this measurement was BLE specific and does _not_
35  completely correct accuracy on any arbitrary PHY configuration. For custom
36  PHY configurations you will still have to manually calibrate out any offsets
37  by measurement and any previous measurements are likely different now.
38  * Moved BER test implementation out of the RAIL library and into RAILTest.
39  Deprecated the existing BER test functions in the RAIL API. With BER test
40  code now existing in RAILTest, you must use version 2.2.1 or later of the
41  radio configurator.
42  * Disable receive buffer overflow and transmit buffer underflow as
43  BUFC_IRQHandler() events. These events now only occur in FRC_IRQHandler().
44  * Speed up RAIL transmit and receive buffer read/write operations.
45  * Fix a race condition that could cause the receive packet callback to be
46  delayed until the next receive packet event.
47 
48 ### 1.5.0
49  * Added other methods of \ref Data_Management. The application can
50  configure RAIL to provide/retrieve data via FIFO method.
51  * The new FIFO method of data management brings with it two new callbacks:
52  RAILCb_TxFifoAlmostEmpty(), and RAILCb_RxFifoAlmostFull(). Note that you
53  must implement these new callbacks or stub them out if you do not intend to
54  use them.

56 ### 1.4.2
57  * Fixed a bug in how we handled IEEE 802.15.4 data request packets.
58  Specifically, we were not issuing the
59  RAILCb_IEEE802154_DataRequestCommand() callback for data request packets
60  that were sent with MAC security enabled.
61  * Fixed a bug where calling RAIL_TxDataLoad() at an inopportune time could
62  cause that and all subsequent such calls to fail.
63 
64 ### 1.4.1
65  * Implemented LQI measurement. The field in RAIL_AppendedInfo_t is now
66  populated on every packet receive.
67  * Allow transmit while receive is active, if the channel is the same. After
68  transmitting, the radio will follow the configured state transitions as
69  before.
70  * Allow shorter ccaBackoff times if receive is active during the CSMA or LBT
71  process.
72  * Allow calling RAIL_RxConfig while receiving.
73  * Added new receive event: RAIL_RX_CONFIG_PACKET_ABORTED
74  * Added new RAIL_TxConfig API, along with more transmit events:
75  RAIL_TX_CONFIG_CHANNEL_CLEAR, RAIL_TX_CONFIG_CCA_RETRY,
76  RAIL_TX_CONFIG_START_CCA
77  * Fixed a filtering bug in the IEEE 802.15.4 code. We were incorrectly filtering
78  packets with a 0xFFFF source PANID while the coordinator which caused
79  problems in the joining process. The new logic will only use the source
80  PANID for filtering if we are the coordinator, there is no destination
81  address, and the frame type is MAC command or Data.
82  * Migrated to Gecko SDK 5.0, including the new em_core APIs for interrupt
83  enabling and disabling. This adds required dependencies on the
84  CORE_EnterCritical() and CORE_ExitCritical() functions.
85  * Added new RSSI functions for hardware and software averaging:
86  RAIL_StartAverageRSSI(), RAIL_GetAverageRSSI(), RAIL_AverageRSSIReady(),
87  RAIL_PollAverageRSSI().
88  * Added new RSSI callback: RAILCb_RssiAverageDone().
89  * Added the ability to switch between multiple BLE phys, including 2Mbps phys,
90  on chips that support the new phys.
91 
92 ### 1.4.0
93  * Added support for the EFR32MG12P part family with larger RAM and Flash size
94  variants along with some new chip features.
95  * Renamed anything that was just efr32 in the previous releases to efr32xg1 to
96  differentiate it from the new efr32xg12 family of parts.
97  * Add support for the new BLE 2MBit PHY to the BLE acceleration and
98  configuration APIs on EFR32xG12 chips.

100 ### 1.3.2
101  * Added RAIL_SetPtiProtocol() and \ref RAIL_PtiProtocol_t. This function
102  allows the application to configure RAIL's packet trace interface to output
103  protocol specific data.
104  * Fix BER functions to operate at 1Mbps data rates.
105  * RAIL_RfIdle() does not clear the transmit buffer anymore. RAIL_TxDataLoad()
106  resets the transmit buffer with every packet set. This fixes the case where
107  RAIL_RfIdle() is called after RAIL_TxDataLoad().
108  * Fixed bug when using fixed packet lengths configured via the Radio
109  Configurator. RAIL_TxStart() will not overwrite the configured fixed
110  length to the amount of data loaded by RAIL_TxDataLoad().
111  * Fixed bug in RADIO_PA_Init() where ramp times greater than 11.2 ms would be
112  truncated to a very short ramp.
113 
114 ### 1.3.1
115  * Added IEEE 802.15.4 hardware acceleration. The new APIs can be found in
116  protocol/ieee802154/rail_ieee802154.h. Documentation for these APIs can be
117  found in \ref IEEE802_15_4. This includes a new callback:
118  RAILCb_IEEE802154_DataRequestCommand().
119  * Added Auto Ack hardware acceleration. Documentation for these APIs
120  can be found in \ref Auto_Ack.
121  * Added RAIL_TxStartWithOptions(). This new API takes a pointer to \ref
122  RAIL_TxOptions_t that contains options to modify the upcoming transmit.
123  * Increased minimum state transition time to 100 us.
124  * Added \ref RAIL_STATUS_INVALID_CALL
125  * Added \ref RAIL_AppendedInfo_t::isAck
126 
127 ### 1.3.0
128  * Added BLE hardware acceleration. The new APIs can be found in
129  protocol/ble/rail_ble.h. Documentation for these APIs can be found in \ref
130  BLE.
131  * Added RAIL_ScheduleRx() and \ref RAIL_ScheduleRxConfig_t. This allows the
132  application to schedule a RX window using the RAIL timebase.
133  * Added \ref RAIL_TIME_DISABLED enum to \ref RAIL_TimeMode_t.
134  * Added an extended idle function which gives you more control of how the
135  radio is idled. See RAIL_RfIdleExt() and \ref RAIL_RfIdleMode_t for more
136  details.
137  * Added \ref RAIL_RX_CONFIG_TIMEOUT and \ref RAIL_RX_CONFIG_SCHEDULED_RX_END
138  * Added \ref RAIL_TX_CONFIG_TX_ABORTED and \ref RAIL_TX_CONFIG_TX_BLOCKED
139  * Added an extended radio status callback that supports up to 32 different
140  status reasons (RAILCb_RxRadioStatusExt()). You may implement this
141  instead of RAILCb_RxRadioStatus() or continue using the old version if
142  you do not need access to more than the first 8 status values.

160  * Optimized address filtering code for operation at higher data rates.

163 ### 1.2.2
164  * Added RAIL_SetTime() in order to allow the user to change the RAIL
165  timebase with microsecond granularity.
166  * Removed sprurious calls to RAIL_RxRadioStatus() with the \ref
167  RAIL_RX_CONFIG_BUFFER_OVERFLOW argument, which were happening when receive
168  was aborted due to events such as address filtering failures.
169  * Fixed incorrect PA output power calculations.
170 
171 ### 1.2.1
172  * Allow RAIL_RxStart() and RAIL_TxStart() to not error if the radio is
173  heading to idle after a call to RAIL_RfIdle(). Application code does not
174  have to wait until the radio is completely idle before calling a receive or
175  transmit operation as long as the radio is heading to idle.

177 ### 1.2.0
178  * Fixed IR calibration such that calibration values are more accurate for
179  radio configurations with datarates below 10Kbps.
180  * Fixed IR calibration such that the default calibration value is applied
181  when a better calibration value is unable to be calculated.
182  * Added RAIL_CalInit() for calibration initialization.
183  * Added BER test API and structures for diagnostic use -
184  RAIL_BerConfigSet(), RAIL_BerRxStart(), RAIL_BerRxStop(),
185  RAIL_BerStatusGet(), \ref RAIL_BerConfig_t, \ref RAIL_BerStatus_t.
186  * Enabled DC Calibration during initialization.
187 
188 ### 1.1.1
189  * Fixed CCA timing to prevent invalid CCA failures when starting out of IDLE
190  state.
191  * Fixed RX state transitions to TX when ignoring CRC errors. Will properly take
192  the success route when a CRC error occurs while ignoring CRC errors.
193  * Fixed TX error state transition after CCA failure.
194 
195 ### 1.1.0
196  * RAIL_RX_CONFIG_INVALID_CRC changed to \ref RAIL_RX_CONFIG_FRAME_ERROR
197  * This now accurately represents the callback flag.
198  * A frame error is either an invalid crc, a frame coding error, or a
199  variable length error. On detection of a frame error, the radio will
200  proceed to the defined error state.
201  * RAIL_SetRxTransitions() and RAIL_SetTxTransitions() APIs added
202  * Add ability to configure radio to automatically to transition to a
203  given radio state on completion of a receive or transmit event.
204  * RAIL_SetStateTiming() API added
205  * State transition times can be configured. The given timestamps will not
206  perfectly match on-air times, due to the RX and TX chain delays, which
207  are dependent on the radio configuration.
208  * RAIL_AddressFilterByFrameType() added
209  * If address filtering and frame type length decoding are both enabled,
210  this gives the ability to enable or disable address filtering per frame
211  type
212 
213 ### 1.0.0
214  * Return quarter dBm resolution from RAIL_RxGetRSSI(). This changes the return
215  value to an int16_t from an int8_t.
216  * Added calibration APIs (\ref Calibration)
217  * The RAIL library will notify the application via callback,
218  RAILCb_CalNeeded(), whenever it detects a calibration is needed.
219  * An API is provided allowing the application to choose when to calibrate
220  the radio, to pass in a known calibration value, or force a calibration
221  to occur.
222  * RAIL_TxPowerSet() returns the power level that can be set which might not
223  match the requested power level.
224 
225 ### Beta 1 - November 16, 2015
226  * Exposed EFR32 RF Energy Sensing capability via a RAIL API.
227  * This functionality allows the radio to be woken from sleep by a large
228  amount of energy over the air.
229  * Added new RAIL Timer APIs (\ref System_Timing)
230  * This provides access to a hardware timer running at the RAIL timebase. You
231  can use this to implement simple protocol timing that may be needed by
232  your low level radio code.
233  * This change also adds a new RAILCb_TimerExpired() that must be implemented
234  by your application.
235  * Added APIs to access the bit rate and symbol rate for the current PHY
236  (RAIL_BitRateGet(), RAIL_SymbolRateGet()).
237  * Cleaned up RAIL type names so that they all begin with the RAIL_ prefix.
238  This will require updates to existing application code.
239  * Initial integration with Simplicity Studio's application builder tools
240 
241 ### Alpha 2 - September 14, 2015
242  * Filtering of receive packets based on defined address fields and compare
243  values. See \ref Address_Filtering for more details.
244  * Added different \ref Pre-Transmit options for the Tx function.
245  * Scheduling of packet transmission at an absolute time or after a defined
246  delay.
247  * Packet transmission conditional on Clear Channel Assessment (CSMA, LBT).
248  * The ability to read RSSI manually with RAIL_RxGetRSSI().
249  * Support for building RAIL applications using both GCC and IAR toolchains.
250 
251 ### Alpha 1
252  * Initial release with support for transmit receive and basic radio
253  configuration.
