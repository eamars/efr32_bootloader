66 // The basic TOKEN_DEF macro should not be used directly since the simplified
67 // definitions are safer to use. For completeness of information, the basic
68 // macro has the following format:
69 //
70 // TOKEN_DEF(name,creator,iscnt,isidx,type,arraysize,...)
71 // name - The root name used for the token
72 // creator - a "creator code" used to uniquely identify the token
73 // iscnt - a bool flag that is set to identify a counter token
74 // isidx - a bool flag that is set to identify an indexed token
75 // type - the basic type or typdef of the token
76 // arraysize - the number of elements making up an indexed token
77 // ... - initializers used when reseting the tokens to default values
78 //
79 //
80 // The following convenience macros are used to simplify the definition
81 // process for commonly specified parameters to the basic TOKEN_DEF macro
82 // DEFINE_BASIC_TOKEN(name, type, ...)
83 // DEFINE_INDEXED_TOKEN(name, type, arraysize, ...)
84 // DEFINE_COUNTER_TOKEN(name, type, ...)
85 // DEFINE_FIXED_BASIC_TOKEN(name, type, address, ...)
86 // DEFINE_FIXED_INDEXED_TOKEN(name, type, arraysize, address, ...)
87 // DEFINE_FIXED_COUNTER_TOKEN(name, type, address, ...)
88 // DEFINE_MFG_TOKEN(name, type, address, ...)
89 //
