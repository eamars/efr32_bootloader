void emberAfPluginDeviceTableMessageReceived ( EmberNodeId  nodeId)

Inform the device table we have received a message.

The device table keeps track of whether a device has become unresponsive. Hearing from a known device will cause the device table to transition the device state from "unresponsive" to "joined". Hearing from an unknown device will cause the device table to discover the device, potentially fixing routing issues (i.e. if the node ID has changed).

Parameters
nodeIdNode ID from which the last incoming message came.
void emberAfPluginDeviceTableInitiateRouteRepair ( EmberNodeId  nodeId)

Kick off device table route repair.

We have sent a message to the device and it failed. Therefore, we need to attempt route repair and mark the node as unresponsive. Currently, route repair simply sends a broadcast message looking for the node ID based on the EUI64. We may replace with something more sophisticated in the future.

Parameters
nodeIdNode ID of the newly unresponsive node.

void emberAfDeviceTableNewDeviceJoinHandler ( EmberNodeId  newNodeId,
EmberEUI64  newNodeEui64 
)

Inform the device table that a new device joined.

When a device joins, rejoins, or sends an end-device announce, we need to call this function in the device table. If the device is new, it will generate a new device table entry and kick off the discovery process. If the device is not new, the device-table will make sure the node ID matches the EUI64 and make an update if necessary. It will also check to verify the discovery process has completed, and attempt to complete it.

Parameters
newNodeIdNode id of the newly joined device.
newNodeEui64EUI64 of the newly joined device
