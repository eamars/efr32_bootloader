Silicon Labs EmberZNet Documentation

Release Notes / Quick Start Guide

  • EmberZNet PRO Release Notes

    EmberZNet: Lists compatibility requirements and sources for all software components in the development environment. Discusses the latest changes to the EmberZNet PRO stack (and associated utilities) including added/deleted/deprecated features/API, and lists bugs that have been fixed since the last release and any pending ones.

  • Application Framework V2 for EmberZNet PRO Release Notes

    EmberZNet: Discusses the latest changes to the AFV2 (application framework V2) including added/deleted/deprecated features/API. Details any additions to Zigbee Application Profiles and discusses corresponding changes to the framework. Lists bugs that have been fixed since the last release and any pending ones.

  • Getting Started with EmberZNet PRO -- QSG106

    EmberZNet: Provides basic information on configuring, building, and installing applications for the Mighty Gecko family and EM35x using the EmberZNet PRO stack and Simplicity Studio v4.

  • Getting Started with the EM34x Development Kit -- QSG100

    EmberZNet: Contains a storyboard-like description of the Ember EM34x Low-Cost Remote Development Kit's contents, and provides step-by-step instructions on setting up the hardware and software.

Zigbee Fundamentals

  • Fundamentals - Zigbee -- UG103-02

    EmberZNet: Describes the key features and characteristics of a zigbee solution. It also includes a section on zigbee 3.0.

API References

  • API Reference for the EmberZNet SOC Platform

    EmberZNet: Lists SoC Platform APIs used to interface to the EmberZNet PRO stack, HAL, and status of the application-controlled network. These APIs concern network management, device and stack management, messaging, fragmentation, serial communication, token access, peripheral access, bootload utilities, and others. They are independent of the Application Framework and therefore can be used to develop applications that do not rely on the Zigbee Cluster Library.

  • API Reference for the EmberZNet STM32F103RET Host

    EmberZNet: Lists STM32F103RET Host APIs used to interface to the EmberZNet PRO stack, HAL, and status of the application-controlled network. These APIs concern network management, device and stack configuration, message fragmentation, serial communication, peripheral access, and others. They are independent of the Application Framework and therefore can be used to develop applications that do not rely on the Zigbee Cluster Library.

  • API Reference for EmberZNet PC Host

    EmberZNet: Lists PC Host APIs used to interface to the EmberZNet stack, HAL, and status of the application-controlled network. These APIs concern network management, device and stack configuration, message fragmentation, serial communication, ASH (Asynchronus Serial Host) utilities, and others. They are independent of the Application Framework and therefore can be used for developing applications that do not rely on the ZigBee Cluster Library.

  • API Reference for EmberZNet RF4CE

    EmberZNet: Lists RF4CE APIS used to interface to the stack, HAL, and application utilities.

  • Advanced Application Programming with the Stack and HAL APIs -- UG105

    EmberZNet: A companion to the EmberZNet API references, for developers whose applications require functionality not available through AppBuilder and the application framework, or who prefer working with an API. Includes an introduction to the stack API, a discussion of advanced design issues to consider when developing an application using the API, and provides an example application.

Application Framework

  • Application Framework V2 (AFV2) API

    EmberZNet: Describes AFV2 APIs, the CLI interface, and callbacks.

  • RF4CE Application Framework Reference

    EmberZNet: Lists APIs used to implement RF4CE standards. Also describes a CLI, example applications, and architecture-agnostic plugins.

  • Application Framework V2 (AFV2) Developer Guide -- UG102

    EmberZNet: Covers the AFV2 structure and usage. Topics include the callback interface, time handling, event management, attributes management, multinetwork management, plugin management, sleepy Device management, and ZCL command handling.

Application Development Fundamentals

  • Fundamentals - Wireless Networking -- UG103-01

    EmberZNet: Introduces some fundamental concepts of wireless networking. These concepts are referred to in other Fundamentals documents. If you are new to wireless networking, read this document first.

  • Fundamentals - Design Choices -- UG103-03

    EmberZNet: Discusses the major decisions that must be made about which wireless protocol you should use, as well as additional decisions to be made if you are designing a zigbee solution.

  • Fundamentals - HAL -- UG103-04

    EmberZNet: Silicon Labs HAL (Hardware Abstraction Layer) is program code between a system's hardware and its software that provides a consistent interface for applications that can run on several different hardware platforms. The HAL is designed for developers using EmberZNet PRO, EmberZNet RF4CE, and Silicon Labs Thread on the Wireless Gecko (EFR32) portfolio and the EM3x-Based MCU family of products.

  • Fundamentals - Security -- UG103-05

    EmberZNet: Introduces some basic security concepts, including network layer security, trust centers, and application support layer security features. Discusses the types of standard protocols available in EmberZNet PRO. Coding requirements for implementing security are reviewed in summary. Finally, information on implementing zigbee Smart Energy security is provided.

  • Fundamentals - Bootloading -- UG103-06

    EmberZNet: Introduces bootloading for Silicon Labs networking devices. Discusses the Gecko Bootloader as well as legacy Ember and Bluetooth bootloaders, and describes the file formats used by each.

  • Fundamentals - Tokens -- UG103-07

    EmberZNet: Describes tokens and shows how to use them in EmberZNet PRO, EmberZNet RF4CE, and Silicon Labs Thread applications. It also discusses bindings, the zigbee PRO application-defined associations between two devices on a network.

  • Fundamentals - Tools -- UG103-08

    EmberZNet: Provides an overview of the toolchain used to develop, build, and deploy EmberZNet and Silicon Labs Thread applications, and discusses some addition tools and utilities.

  • Fundamentals - ZLL -- UG103-09

    EmberZNet: Compares the ZLL stack and network with the EmberZNet PRO stack and network, with notes about considerations when implementing a ZLL solution. Includes a basic description of ZLL configuration and commissioning, and notes about the interoperability of ZLL and non-ZLL devices.

  • Fundamentals - RF4CE -- UG103-10

    EmberZNet: Describes the zigbee RF4CE specification, with notes about considerations when implementing an RF4CE solution. Includes a basic description of RF4CE device types, the network formation process, power saving, and security.

Users Guides

  • Testing and Debugging Applications for the Silicon Labs EM35x and Mighty Gecko (EFR32MG) Platforms -- UG104

    EmberZNet: Describes strategies for testing and debugging applications, including: hardware and application considerations, initial development testing, and lab testing. For additional information about later stages of programming and testing see application notes AN700, Manufacturing Test Guidelines, and AN718, Manufacturing Test Overview.

  • EM35x Development Kit User Guide -- UG110

    EmberZNet: Explains how to set up EM35x Development kit hardware and use the Debug Adapter (ISA3). Describes some of the software tools available, along with examples of their use.

  • EM34x User Guide -- UG113

    EmberZNet: Describes the EM34x Low Cost Remote Development Kit and the EM34x Full-Feature Remote Development Kit, and explains how to set up their hardware and use the Debug Adapter (ISA3).

  • ISA3 Utilities Guide -- UG107

    EmberZNet: Describes how to use the standalone ISA3 utilities supplied with EmberZNet PRO and Silicon Labs Thread software. It also gives a brief overview of the various file types that may be used.

  • Simplicity Commander Reference Guide -- UG162

    EmberZNet: Describes how and when to use Simplicity Commander's Command-Line Interface. Only EFR32 is currently supported. EM3xx is not supported at this time.

  • EZSP Protocol Reference Guide -- UG100

    EmberZNet: Describes the EmberZNet Serial Protocol (EZSP), used by a host application processor to interact with the EmberZNet PRO stack running on an NCP over either a SPI or a UART interface. Describes the frame formats for different EZSP-bound stack activities such as network management, messaging, bootloading, and token access.

  • UART Protocol Reference Guide -- UG101

    EmberZNet: Describes the protocol used by EZSP-UART to reliably carry commands and responses between a host processor and a network co-processor. The topics discussed include a brief overview of Ember-designed ASH (Asynchronus Serial Host) protocol, general ASH frame format, different ASH frames, and their oprations.

  • Silicon Labs Gecko Bootloader User's Guide -- UG266

    EmberZNet: Describes the high-level implementation of the Silicon Labs Gecko Bootloader for EFR32 SoCs and NCPs, and provides information on how to get started using the Gecko Bootloader with Silicon Labs wireless protocol stacks.

  • Multiprotocol User's Guide -- UG267

    EmberZNet: Provides details about the Silicon Labs multiprotocol implementation released in March 2017, including details on the Multiprotocol Stack Interface (MPSI) and data storage issues. Also provides a detailed procedure for using the switched multiprotol feature.

Application Notes

  • Manufacturing Test Overview -- AN718

    EmberZNet: Provides a high-level description of the different options for integrating RF testing and characterization into your standard test flows. It is intended for customers who are moving from the early prototype development stage to the manufacturing production environment and need assistance with manufacturing test.

  • Manufacturing Test Guidelines -- AN700

    EmberZNet: Details the different options for integrating RF testing and characterization into your standard test flows.

  • Setting Manufacturing Certificates and Installation Codes -- AN708

    EmberZNet: Provides an overview of the use of install codes and certificates in a Smart Energy network. Describes the components of a Smart Energy certificate and the differences between test certificates and production certificates. Explains how to use Silicon Labs utilities to program, verify and erase install codes and certificates.

  • Bringing Up Custom Devices for the EM35x SoC or NCP Platform -- AN710

    EmberZNet: Contains a list of Customer Information Block (CIB) tokens and explains how to program/use the CIB, a special non-volatile area of Flash that contains information such as custom EUI64, install code, certificates and other data that can only be programmed through an external interface such as JTAG serial wire.

  • Programming Options - For the EM35x Platform -- AN717

    EmberZNet: Describes the various programming options available to EM35x series customers - the Debug Adapter (ISA3) for the developer environment and the prototype and low-volume production environment, and gang programmers, in-system programmers (ISPs), and in-circuit test (ICT) programming options for the high-volume production environment.

  • Using the Simulated EEPROM for the EM35x and Mighty Gecko Family SoC Platforms -- AN703

    EmberZNet: Describes how to use the simulated EEPROM in Silicon Labs' EM35x and Mighty Gecko (EFR32MG) platforms, gives an overview of the internal design, and discusses design considerations. You should already be familiar with the token system in the HAL. For more information on HAL and the token system, see the online HAL API documentation.

  • EZSP-UART Host Interfacing Guide -- AN706

    EmberZNet: Describes procedures for initial tests of a host connected to a zigbee processor using EZSP-UART. It assumes that you have already read UG101, the UART Gateway Protocol Reference Guide. You should have a basic understanding of the UART Gateway protocol, as well as the signals needed by the UART interface.

  • Measuring EM35x Power Consumption For Sleepy End Devices using EmberZNet -- AN713

    EmberZNet: Describes techniques for making detailed measurements of the power consumption of sleepy end devices running EmberZNet PRO on the EM35x SOC platform. Includes information on test setup and some common usage scenarios, besides providing details about what operations are observed in the measurements.

  • Smart Energy ECC-Enabled Device Setup Process -- AN714

    EmberZNet: Describes how to set up a device with the security resources required to support Smart Energy (SE) security, which is based on certificate-based key establishment (CBKE) using Elliptic-Curve Cryptography (ECC). You should be familiar the Zigbee Smart Energy Profile specification.

  • Instructions for Using Image Builder -- AN716

    EmberZNet: Provides instructions for creating zigbee Over-the-air (OTA) bootloader files with Image Builder, which takes an existing file (or multiple files) and wraps them in the file format as declared in the zigbee specification.

  • Adjacent Channel Rejection Measurements - For 802.15.4 radios -- AN709

    EmberZNet: Presents the method Silicon Labs uses to measure Adjacent channel rejection (ACR) on its IEEE 802.15.4-2003 compliant ICs, and compares it to other methods. ACR is a measure of how well a reciever performs on its frequency channel when there is an interfering system in the vicinity operating on a nearby channel.

  • Using the EM35x ADC -- AN715

    EmberZNet: Describes how to use the EM35x Analog to Digital Converter (ADC), a first-order sigma-delta converter, and also gives an overview of the internal design, discusses design considerations, and provides example code of fundamental usage.

  • Using the Ember EM358x/EM359x USB -- AN740

    EmberZNet: Provides key information for using the Ember EM358x USB. The goal of the EM358x USB is to only act as COM port replacement for UART-style functionality.

  • Using the Ember Standalone Bootloader -- AN760

    EmberZNet: For EFR32xG12 platforms and newer, see UG266: Silicon Labs Gecko Bootloader User Guide. This describes using the legacy Ember Standalone Bootloader for EM3x and early EFR32 platforms. The Ember Standalone bootloader is a simple, dedicated program to facilitate input of new application/stack firmware via Xmodem upload to a serial interface (SPI or UART) or via a proprietary, IEEE 802.15.4-based, single-hop MAC layer RF protocol.

  • Using the Ember Application Bootloader -- AN772

    EmberZNet: For EFR32xG12 platforms and newer, see UG266: Silicon Labs Gecko Bootloader User Guide. This describes using the legacy Ember Application Bootloader for EM3x and early EFR32 platforms. The Ember application bootloader is stored on the chip along with the stack/application, and facilitates firmware image updates by the application.

  • Zigbee OTA Bootload Cluster Server and Client Setup -- AN728

    EmberZNet: Describes how to perform a zigbee over-the-air (OTA) bootload cluster download between a client and server device. The instructions are for Smart Energy 1.x devices using Ember EM35x development kits. Users can also refer to this procedure when setting up or testing zigbee OTA bootload cluster download in their own development environments with their own hardware.

  • Designing for Multiple Networks on a Single Zigbee Chip -- AN724

    EmberZNet: Describes the multi-network stack feature that allows a single-radio chip to be concurrently part of more than one distinct network. Some limitations and restrictions are enforced by the multi-network stack and should be taken into account during the design of a multi-network application. These limitations are mostly related to the role the node assumes on the networks and are discussed here in detail.

  • Application Framework Command Line Interface for RF4CE -- AN879

    EmberZNet: Describes the command line interface (CLI) functionality provided in the Application Framework for RF4CE-based applications.

  • RF4CE GDP Enhanced Security -- AN875

    EmberZNet: Describes the enhanced security feature (based on AES-CMAC) available for use with RF4CE.

  • RF4CE Zigbee Remote Control Firmware Guide -- AN906

    EmberZNet: Explains Silicon Labs RF4CE ZRC (Zigbee Remote Control) Low Cost Remote and Full Featured Remote reference designs main firmware structures and how to customize them.

  • EZSP-SPI Host Interfacing Guide -- AN711

    EmberZNet: Details the EZSP-SPI Protocol used by a host microcontroller to communicate with an Ember network co-processor (NCP) running the EmberZNet PRO stack. It includes recommended procedures for developing and testing a driver for the EZSP-SPI Protocol on a new host microcontroller.

  • Bringing Up New Microcontrollers for an NCP and EmberZNet PRO 5.x -- AN712

    EmberZNet: Describes recommended practices and procedures for designing, developing, and testing the SPI Protocol on a new microcontroller that is interfaced to a network co-processor (NCP) and uses the EmberZNet PRO stack software. Discusses physical interfaces between the host and the co-processor, initialization of SPI transacations, and extended communication. You should understand the basics of EZSP (refer to the EZSP reference guide) and the EM35x data sheet.

  • Bringing Up Custom Devices for the Mighty Gecko and Flex Gecko Families -- AN961

    EmberZNet: Describes how to initialize a piece of custom hardware (a 'device') based on the Mighty Gecko and Flex Gecko families so that it interfaces correctly with a network stack. The same procedures can be used to restore devices whose settings have been corrupted or erased.

  • Building a Customized NCP Application -- AN1010

    EmberZNet: Provides instructions for configuring various aspects of an NCP application image using the Application Builder tool within Simplicity Studio.

  • Using the NodeTest Application -- AN1019

    EmberZNet: Describes how to use NodeTest, a pre-built application supplied by Silicon Labs for the purpose of performing RF evaluation, functional testing, and hardware validation. It contains RF test functions pertinent to IEEE 802.15.4-based radio configurations such as those used by the Silicon Labs EmberZNet PRO and Silicon Labs Thread stacks on platforms such as the EM35x, Mighty Gecko, or Flex Gecko families.

  • Zigbee and Thread Coexistence with WiFi -- AN1017

    EmberZNet: Describes the impact of Wi-Fi on zigbee and Thread, and methods to improve coexistence. First, methods to improve coexistence without direct interaction between zigbee/Thread and Wi-Fi radios are described (for both EM35x/EM358x and EFR32MG). Second, Silicon Labs's Packet Traffic Arbitration (PTA) support to coordinate 2.5 GHz RF traffic for co-located zigbee/Thread and Wi-Fi radios is described (for the EFR32MG only).
